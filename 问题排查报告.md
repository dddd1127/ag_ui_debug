# AG-UI Debug 项目问题排查报告

## 🔴 严重问题（必须修复，否则项目无法运行）

### 问题 1：agentscope 2.0 API 不兼容 — `ReActAgent` 不存在

**文件**: `server/agent_runner.py:29`

**现状**:
```python
from agentscope.agent import ReActAgent
```

**问题**: agentscope 2.0 中 `ReActAgent` 已被移除。现在只有一个统一的 `Agent` 类，ReAct 行为通过 `ReActConfig` 配置。

**正确 API**:
```python
from agentscope.agent import Agent, ReActConfig

agent = Agent(
    name="Friday",
    system_prompt="You are a helpful assistant named Friday.",
    model=...,  # 需要 OpenAICredential
    toolkit=toolkit,
    react_config=ReActConfig(max_iters=20),
)
```

### 问题 2：`AnthropicChatModel` 构造函数签名完全不同

**文件**: `server/agent_runner.py:87-93`

**现状**:
```python
model=AnthropicChatModel(
    model_name,
    api_key=api_key,
    client_kwargs={"base_url": base_url},
    stream=True,
)
```

**问题**: agentscope 2.0 的 `AnthropicChatModel` 构造函数签名变为：
```python
AnthropicChatModel(
    credential: AnthropicCredential,  # 新增：需要 credential 对象
    model: str,                       # 参数名从 model_name 改为 model
    parameters: Parameters | None,
    stream: bool,
    ...
)
```

而且 `AnthropicCredential` 的字段是 `api_key` + `base_url`，不再是直接传参。

**正确 API**:
```python
from agentscope.credential import OpenAICredential  # 用 OpenAI 兼容模式
# 或
from agentscope.credential import AnthropicCredential

credential = OpenAICredential(
    api_key=api_key,
    base_url=base_url,
)
model = OpenAIChatModel(
    credential=credential,
    model=model_name,
    stream=True,
)
```

### 问题 3：`stream_printing_messages` 不存在

**文件**: `server/agent_runner.py:35, 137`

**现状**:
```python
from agentscope.pipeline import stream_printing_messages
...
async for msg, last in stream_printing_messages(
    agents=[agent],
    coroutine_task=agent(messages),
):
    yield msg, last
```

**问题**: agentscope 2.0 中 `agentscope.pipeline` 模块已不存在。`stream_printing_messages` 这个旧版 `(msg, last)` 元组模式已完全废弃。

agentscope 2.0 使用 **事件流** 模式：
```python
async for event in agent.reply_stream(msgs):
    # event 是 AgentEvent 的子类：
    #   TextBlockStartEvent, TextBlockDeltaEvent, TextBlockEndEvent
    #   ToolCallStartEvent, ToolCallDeltaEvent, ToolCallEndEvent
    #   ToolResultStartEvent, ToolResultEndEvent
    #   ReplyStartEvent, ReplyEndEvent
    #   ...
```

这意味着 `run_agent_stream` 函数的返回值类型完全不同，整个 `agui_endpoint.py` 和 `agui_adapter.py` 的响应转换逻辑都需要重写。

### 问题 6：`AgentRequest.input` 的消息格式与 agentscope 2.0 `Msg` 不兼容

**文件**: `server/agui_adapter.py`, `server/agent_runner.py`

**问题**: `agui_adapter.py` 将 AG-UI 消息转换为 `agentscope_runtime` 的 `AgentRequest.input`（`Message` 列表）。但 `agent_runner.py` 试图将这些直接传给 Agent。

在 agentscope_runtime 框架中，这通过 `message_to_agentscope_msg()` 转换器完成。在独立实现中，你需要自己将 `AgentRequest.input` 中的 `Message` 转换为 agentscope 2.0 的 `Msg` 对象（需要 `name`, `content: list[ContentBlock]`, `role` 字段），而不是直接传递。

---

### 问题 7：缺少 `__init__.py`

**文件**: `server/__init__.py`

**问题**: `server/` 目录缺少 `__init__.py` 文件。虽然 Python 3 支持隐式命名空间包，但当 `main.py` 使用 `from app import create_app` 这样的相对导入时，可能会遇到模块查找问题。建议添加空的 `__init__.py`。

## 🟡 中等问题（功能受影响，但不会直接崩溃）

### 问题 8：`agui_adapter.py` 的响应转换逻辑基于旧版 `(msg, last)` 元组模式

**文件**: `server/agui_adapter.py:392-528`

**问题**: `agent_event_to_agui_events()` 函数处理的是 `Content | Message | BaseResponse` 类型的旧版事件。但 agentscope 2.0 的 `reply_stream()` 返回的是全新的 `AgentEvent` 层级：

- `ReplyStartEvent` / `ReplyEndEvent`
- `TextBlockStartEvent` / `TextBlockDeltaEvent` / `TextBlockEndEvent`
- `ToolCallStartEvent` / `ToolCallDeltaEvent` / `ToolCallEndEvent`
- `ToolResultStartEvent` / `ToolResultTextDeltaEvent` / `ToolResultEndEvent`

你需要重写整个转换链：
1. agentscope 2.0 events → agentscope_runtime events（类似 `adapt_agentscope_message_stream`）
2. agentscope_runtime events → AG-UI events

**或者**，更简单的方案：直接将 agentscope 2.0 events → AG-UI events（跳过中间层）。

### 问题 10：前端 `useAguiSSE.ts` 中 `sendMessage` 依赖过时的 `messages` 状态

**文件**: `web/src/hooks/useAguiSSE.ts:221`

**现状**:
```typescript
const request = buildAguiRequest(text, messages)
```

**问题**: `sendMessage` 是 `useCallback` 依赖了 `messages`，但 `messages` 在 `sendMessage` 内部通过 `setMessages` 更新后，闭包中的 `messages` 仍是旧值。这会导致连续发送消息时，历史消息不完整。

**修复**: 使用 `setMessages` 的函数形式获取最新状态：
```typescript
const sendMessage = useCallback((text: string) => {
  setMessages(prev => {
    const request = buildAguiRequest(text, prev)
    // ... 发送逻辑
    return [...prev, userMsg]
  })
}, [])  // 移除 messages 依赖
```

### 问题 11：前端每次发送消息都生成新的 `threadId`

**文件**: `web/src/hooks/useAguiSSE.ts:121`

**现状**:
```typescript
const threadId = `thread_${Date.now()}`
const runId = `run_${Date.now()}`
```

**问题**: 每次调用 `buildAguiRequest` 都会生成新的 `threadId`，这意味着多轮对话中的消息不会共享同一个会话。后端的 session 管理也无法工作。

**修复**: `threadId` 应该在首次创建后保持不变，只更新 `runId`。

## 🔧 核心修复方案：重写 Agent 运行与事件转换层

### 方案 A（推荐）：利用 agentscope_runtime 框架的内部适配器

不要完全自己实现 Agent 运行，而是复用 `agentscope_runtime` 的 `Runner` + 适配器链：

```python
# 伪代码：利用 Runner 来处理 Agent 运行和事件转换
from agentscope_runtime.engine.runner import Runner
from agentscope_runtime.engine.schemas.agent_schemas import AgentRequest

runner = Runner()
runner.framework_type = "agentscope"

# 1. 用 AG-UI 请求构造 AgentRequest
agent_request = agui_request_to_agent_request(run_agent_input)

# 2. 用 Runner.stream_query 获取 agentscope_runtime 事件流
async for event in runner.stream_query(agent_request):
    # event 是 agentscope_runtime 的 Event/Content/Message
    agui_events = agent_event_to_agui_events(event, tracker)
    for ae in agui_events:
        yield serialize_agui_event(ae)
```

**优点**: 复用了消息转换（`message_to_agentscope_msg`）和事件流适配（`adapt_agentscope_message_stream`），不需要自己处理 agentscope 2.0 的复杂事件类型。

**缺点**: 需要正确初始化 `Runner`，将 Agent 的 init/query 逻辑注册到 Runner。

### 方案 B：直接使用 agentscope 2.0 Agent + 自行转换事件

完全跳过 agentscope_runtime 的 Runner，直接使用 Agent.reply_stream()，然后自己将 agentscope 2.0 事件转换为 AG-UI 事件：

```python
from agentscope.agent import Agent
from agentscope.credential import OpenAICredential
from agentscope.model import OpenAIChatModel
from agentscope.message import Msg, UserMsg, TextBlock
from agentscope.event import TextBlockDeltaEvent, ToolCallStartEvent, ...

# 1. 创建 Agent
agent = Agent(name="Friday", system_prompt="...", model=..., toolkit=...)

# 2. 构造消息
msgs = [UserMsg(content=[TextBlock(text="你好")])]

# 3. 运行并转换
async for event in agent.reply_stream(msgs):
    # 直接将 agentscope 2.0 事件映射为 AG-UI 事件
    if isinstance(event, TextBlockDeltaEvent):
        yield serialize(TextMessageContentEvent(delta=event.delta, ...))
    elif isinstance(event, ToolCallStartEvent):
        yield serialize(ToolCallStartEvent(tool_call_name=event.tool_call_name, ...))
    # ...
```

**优点**: 不依赖 agentscope_runtime 框架，更简单直接。

**缺点**: 需要手动处理所有事件类型映射，包括消息状态追踪（START/END 配对）。
